## 業務フロー（AX伴走パートナー バリューチェーン）

仕事は「線形」ではなく「循環」する。下記のサイクルが事業の価値を生む。

```
┌─────────────────────────────────────────────────────────────┐
│              AX伴走パートナー 業務サイクル                   │
└─────────────────────────────────────────────────────────────┘

  ① [AI情報・企画部] ── 外部AIトレンド収集・新サービス設計
          │
          │ サービスメニュー・差別化ポイント
          ▼
  ② [マーケティング部] ── 思想リーダーシップコンテンツ配信
          │
          │ インバウンドリード・認知形成
          ▼
  ③ [リサーチ部] ── ターゲット企業特定・スコアリング
          │
          │ 優先アプローチリスト
          ▼
  ④ [営業部] ── アウトバウンドメール・商談・クロージング
          │
          │ 受注確定
          ▼
  ⑤ [資料制作部] ── 提案書・支援教材・契約書類作成
          │
          │ 伴走開始（オーナーが実施）
          ▼
  ⑥ [ナレッジ管理部] ── 支援実績・成功事例・ノウハウ蓄積
          │
          │ 知識フィードバック
          └──────────────────────────────→ ① へ戻る
```

## 部門別 起動カデンス（トークン効率設計）

| 部署 | 頻度 | トリガー | アウトプット |
|---|---|---|---|
| 経営管理部 | 週1回（月曜・自動） | 自律実行 | 財務・キャパ・攻守レポート |
| AI情報・企画部 | 月2回（第1・3月曜） | マネージャー | AIツール評価・サービス企画 |
| マーケティング部 | 週1回 | マネージャー | コンテンツ素案2〜3本 |
| リサーチ部 | 必要時 | マネージャー | 候補企業リスト |
| 営業部 | 毎日（2社ルール） | マネージャー | 送付メール・ログ更新 |
| 資料制作部 | 必要時 | マネージャー | 提案書・テンプレート |
| ナレッジ管理部 | 週1回 | マネージャー | ナレッジ更新ドキュメント |

## 部門間 連携フロー

```
AI情報・企画部 ──→ マーケティング部（新サービスをコンテンツに）
AI情報・企画部 ──→ 資料制作部（新サービスを提案書に）
AI情報・企画部 ──→ ナレッジ管理部（評価済みツールを知識として蓄積）
リサーチ部     ──→ 営業部（スコアリング済みリストを渡す）
ナレッジ管理部 ──→ AI情報・企画部（現場フィードバックを渡す）
```

## 各ポジションの権限範囲

### オーナー（人間）
- 単独決定：新規事業の立ち上げ・撤退 / 月額10万超の発注
- 日常業務：CEO AIに指示を出すだけ

### CEO AIエージェント
- 単独決定：統括マネージャーへの委任 / ファイナルチェック合否
- オーナーへ確認：エスカレーション条件に該当する事項

### 経営管理部（CEO直轄）
- 自律実行：週次ビジネス健全性レポート
- freee-mcp read：自律実行可 / write：オーナーエスカレーション必須

### 統括マネージャー
- 単独決定：部門への作業指示 / 品質チェック合否 / 差し戻し（最大2回）

### 各部門エージェント
- 単独実行：指示に基づく成果物作成
- 禁止：CEO・オーナーへの直接連絡 / 未承認での外部発信

## エスカレーション条件一覧

| 条件 | 経路 |
|---|---|
| 月額10万円超の発注・契約 | CEO → オーナー |
| 実在する顧客への初回コンタクト実行 | CEO → オーナー |
| 新サービス価格・プランの正式決定 | CEO → オーナー |
| 本番データの書き込み・削除 | CEO → オーナー |
| 新規事業の立ち上げ・撤退判断 | CEO → オーナー |
| 成果物が2回差し戻し後も改善されない | CEO → オーナー |
| 未収金が月商の30%超 | 経営管理部 → CEO → オーナー |
| 稼働案件が上限超で新規受注依頼あり | 経営管理部 → CEO → オーナー |
