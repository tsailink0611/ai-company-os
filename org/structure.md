# 組織構造 — AI Company OS

## 基本思想

オーナー（人間）はCEO AIエージェントに指示を出す。
CEO AIが統括マネージャーへ委任し、統括マネージャーが各部門へ指示する。
部門の成果物は統括マネージャーが品質チェックし、合格のみCEOへ渡す。
CEOがファイナルチェックし、合格のみオーナーへ報告する。
オーナーへの連絡は「エスカレーション条件に該当する場面」のみ。

経営管理部はCEO直轄で週次ビジネス健全性レポートを発行する。

---

```

    ▼

│  タスク受信・分解・委任           │

└──────┬───────────────────────────┘
       │                    ▲
       │                    │ 週次レポート
       │             ┌──────────────────┐
       │             │  経営管理部       │
       │             │  (bizops-agent)  │
       │             │  財務・キャパ・   │
       │             │  攻守ポジション   │
       │             └──────────────────┘
       │
       ▼
┌──────────────────────────────────┐
│  統括マネージャー (general-manager)│
│  部門への作業指示                 │

       │

    ▼                 ▼                 ▼
[営業部]         [リサーチ部]       [その他部門]
[ナレッジ管理部] [資料制作部]      （プロジェクト固有）
```

## 各エージェントの責任範囲

| エージェント | ファイル | 主な責任 |
|---|---|---|
| CEO AI | `org/agents/ceo-ai.md` | 全体統括・委任・ファイナルチェック・エスカレーション |
| 統括マネージャー | `org/agents/general-manager.md` | 部門指示・品質チェック・差し戻し・CEO承認報告 |
| 経営管理部 | `org/agents/bizops-agent.md` | 財務・キャパ・攻守の週次レポート（CEO直轄） |
| 各部門 | 各プロジェクトの `org/agents/` | プロジェクト固有の成果物作成 |

## 意思決定フロー

### 通常タスク
```
オーナー → CEO（指示受信・委任）
  ↓
統括マネージャー（部門へ作業指示）
  ↓
各部門（成果物作成）
  ↓
統括マネージャー（品質チェック）
  ├─ 不合格 → 部門へ差し戻し（最大2回）
  └─ 合格 → CEOへ承認報告
             ├─ CEO不合格 → 統括マネージャーへ差し戻し
             └─ CEO合格 → オーナーへ完了報告
```

### 週次ビジネスレビュー
```
経営管理部（自律実行）→ 財務・稼働データ取得
  ↓
経営管理部 → CEOへ週次レポート送付
  ↓
CEO → 判断不要なら自律処理
    → エスカレーション条件 → オーナーへ
```

### エスカレーションタスク
```
CEO or 統括マネージャー → エスカレーション条件を検知
  ↓
CEO → オーナーへ（条件・選択肢・推奨案を添えて）
  ↓
オーナー → 判断・指示
  ↓
CEO → 統括マネージャー → 部門へ再委任
```

---

## エスカレーション条件（オーナー連絡が必要）

- 月額10万円超の発注・契約・課金
- 実在する顧客への初回コンタクト・メール送信の実行
- 新しいサービス価格・プラン内容の正式決定
- 本番データの書き込み・削除
- 新規事業の立ち上げ・撤退判断
- 成果物が2回差し戻し後も改善されない

## グローバル定義
CEO AI・統括マネージャー・経営管理部はプロジェクト共通エージェントとしてグローバルに定義済み。
- `~/.claude/agents/bizorg-ceo.md`
- `~/.claude/agents/bizorg-manager.md`
- `~/.claude/agents/bizorg-finance.md`

各部門エージェントはプロジェクト固有として各リポジトリの `org/agents/` に定義。
