# 組織構造 — AI Company OS（AX伴走パートナー版）

> フレームワーク：[AI Company OS](https://github.com/tsailink0611/ai-company-os)

## 基本思想

AX伴走パートナーの事業価値は「情報収集 → 認知 → 開拓 → 提案 → 成約 → 知識蓄積」の
サイクルを回すことで生まれる。各部署はこのサイクルの一部を担う。

オーナーへの連絡はエスカレーション条件に該当する場面のみ。

---

## 組織図（ガバナンス）

```
オーナー（人間）
    │ エスカレーション時のみ
    ▼
CEO AIエージェント [ceo-ai]
    │                     ▲ 週次レポート
    │              [経営管理部 bizops-agent]
    ▼              財務・キャパ・攻守判定
統括マネージャー [general-manager]
    │
    ├── [AI情報・企画部] ai-strategy-agent  月2回：AIツール評価・サービス企画
    ├── [マーケティング部] marketing-agent  週1回：コンテンツ素案
    ├── [リサーチ部] research-agent        必要時：候補社調査
    ├── [営業部] sales-agent              毎日：アウトバウンド
    ├── [資料制作部] materials-agent       必要時：提案書・教材
    └── [ナレッジ管理部] knowledge-agent   週1回：ナレッジ更新
```

## 意思決定フロー

### 通常タスク
```
オーナー → CEO（指示受信・委任）
  ↓
統括マネージャー（部門へ作業指示）
  ↓
各部門（成果物作成）
  ↓
統括マネージャー（品質チェック）
  ├─ 不合格 → 部門へ差し戻し（最大2回）
  └─ 合格 → CEOへ承認報告
             ├─ CEO不合格 → 統括マネージャーへ差し戻し
             └─ CEO合格 → オーナーへ完了報告
```

### 自律実行タスク（トリガー不要）
```
経営管理部（毎週月曜）→ CEOへ週次レポート
AI情報・企画部（月2回）→ 統括マネージャーへ月次レポート
マーケティング部（週1回）→ 統括マネージャーへコンテンツ提案
```

### エスカレーションタスク
```
CEO or 経営管理部 → エスカレーション条件を検知
  ↓
CEO → オーナーへ（条件・選択肢・推奨案を添えて）
```

---

## エスカレーション条件（オーナー連絡が必要）

- 月額10万円超の発注・契約・課金
- 実在する顧客への初回コンタクト・メール送信の実行
- 新しいサービス価格・プラン内容の正式決定
- 本番データの書き込み・削除
- 新規事業の立ち上げ・撤退判断
- 成果物が2回差し戻し後も改善されない
- 未収金が月商の30%超（経営管理部経由）
- 稼働案件が上限超で新規受注依頼あり（経営管理部経由）

## グローバル定義（~/.claude/agents/ にコピーして使う）
- `bizorg-ceo.md` — CEO AIエージェント
- `bizorg-manager.md` — 統括マネージャー
- `bizorg-finance.md` — 経営管理部
- `bizorg-ai-strategy.md` — AI情報・企画部
- `bizorg-marketing.md` — マーケティング部

各部門エージェント（リサーチ・営業・資料制作・ナレッジ管理）はプロジェクト固有として
各リポジトリの `org/agents/` に定義。
